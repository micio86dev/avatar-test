# Quality Gates

A task is not complete until its gates pass. Before completing any work:

- Specification updated
- Tests passing
- Coverage requirements satisfied
- Lint passing
- Build successful
- Documentation updated
- No duplicated code
- No dead code
- Security reviewed

## When applicable

For the relevant project type, also verify:

- Accessibility reviewed (see references/accessibility.md)
- Performance / Core Web Vitals reviewed (see references/performance.md)
- Internationalization respected — no new hardcoded user-facing strings (see references/i18n.md)
- SEO and discoverability intact (see references/seo.md)
- Environment-appropriate indexing configured (see references/environments.md)
- Generated HTML/CSS is W3C compliant where reasonable
